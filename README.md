
# 引言：

初衷：写于2018年的12月，广州这座城市依旧几次入冬失败，毕业已近半年。

我的身份是一名前端。未来三年左右大概率仍然将会是一名前端。**在其位、谋其职**，这是基本的工作素养。

前端技术特点：

* 入门易吃透难
* 更新快迭代快
* 内容多内容杂

如果你相信未来是**大数据的未来**，那么对于数据的**整理、检索、运用**是至关重要的。**前端开发技术也是数据！**

开发Debug流程：
* 发现问题
* 确认问题
* 定位问题
* 解决思路
* 具体解决
* 检验测试







作为一个前端攻城狮，需要不断的积累总结和更新知识点，在这个过程中如果能对某个具体的知识点有系列的链接，不用再网上去筛选，而是有一个固定的地方可以查阅、搜索，那必然减少很多筛选的成本。在这里，好的链接被保存，经典的例子被保存，如果可以，进行一个某知识点排名。组件化开发的思想。代码规则那么多，终归是记不了那么多的，所以有快速定位、查找到相关知识点事十分高效的，这少不了平时的积累！你应该明白筛选的重要性！总是有人会去做整理！应该锻炼的不是外包的那种思维，而是产品的思维。记录记录、保持保持。

关键词：前端、知识架构、链接收集、经典问题、知识点更新

>愿望：这个项目成为github上satr数最多的项目（不是倒数第一^_^）。
